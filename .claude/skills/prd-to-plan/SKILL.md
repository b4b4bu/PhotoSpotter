# PRD to Plan

Break a PRD into a phased implementation plan using vertical slices (tracer bullets).

## Process

1. **Confirm your PRD** — Make sure it's in our conversation
2. **Explore your codebase** — Understand current architecture and patterns
3. **Identify durable decisions** — Routes, schema, data models, auth approach
4. **Draft vertical slices** — Each phase cuts end-to-end through all layers, not just one layer
5. **Get your feedback** — Present the breakdown and iterate until you're happy
6. **Write the plan** — Save a structured Markdown file to `./plans/`

## To get started

- Do you have a PRD ready to share, or would you like to paste one?
- If you have an existing codebase, I can explore it to understand your current setup.
